Plakar Open Source Community Code of Conduct

The Plakar community is committed to providing a welcoming, respectful, and inclusive environment for everyone. We value contributions from all members of the community and are dedicated to ensuring that every participant can collaborate openly and effectively. This Code of Conduct outlines our expectations for behavior and guides our commitment to an open, collaborative, and respectful space.

You can read the full, detailed version of this Code of Conduct on our GitHub repository .

Our Standards

  • Be Respectful: Treat others with kindness, empathy, and respect.
  • Be Constructive: Provide helpful feedback, avoid personal attacks.
  • Focus on Technical Merit: Evaluate based on project goals, not bias.
  • Communicate Clearly: Use clear language, disclose conflicts of interest.
  • Foster a Welcoming Environment: Include and support newcomers.
  • Respect Personal Boundaries: Avoid unwelcome or inappropriate behavior.
  • Keep It Professional: No offensive or discriminatory language or imagery.
  • Accept Responsibility: Apologize sincerely and learn from mistakes.

Unacceptable Behavior

  • Harassment, bullying, or discrimination.
  • Insults, threats, or personal attacks.
  • Unsolicited sexual attention or advances.
  • Trolling, spamming, or disruptive behavior.
  • Publishing private information without consent.
  • Making false accusations or misleading statements.

Reporting & Enforcement

Report violations confidentially to conduct@plakar.io. Consequences may include warnings, temporary or permanent suspension, or bans from all project spaces.

Scope

This Code applies to all project spaces, both online and offline, including issue trackers, mailing lists, code repositories, and events.

Contributing to Plakar

We welcome contributions of all kinds from new features and bug fixes to documentation improvements. Your input helps Plakar grow and improve for the entire community. Below is a quick overview of how to get involved. For full details, please read our Contributing Guide .

Ways to Contribute

  • Report bugs or suggest improvements by opening an issue on GitHub.
  • Propose and discuss new features before starting development.
  • Submit code changes via pull requests, following our coding style and review process.
  • Improve or expand the documentation for users and developers.

Best Practices

  • Fork the repository and work on a dedicated feature branch.
  • Write clear commit messages and include relevant tests when applicable.
  • Run tests and linters before submitting your changes.
  • Respect our Code of Conduct in all interactions.

Community vs Enterprise

To fulfill our mission of setting a new open-source standard for data protection, Plakar must ensure the means to develop, maintain, and improve this standard, and the tools that go with it, over time. Securing sustainable funding for the project and our development team is an essential part of that mission and the purpose of Plakar Korp.

Plakar Community Edition will continue to be actively developed and maintained to meet the needs of:

  • Individuals using Plakar for personal purposes
  • Small organizations with simple backup and restore requirements

Plakar Enterprise Edition provides advanced capabilities and will remain freely available for:

  • Non-profit organizations
  • Startups under 3 years old and pre‑Series A
  • Organizations operating in war zones that require assistance

The main focus of the Enterprise Edition is on control plane features, including: multi‑Kloset management with fine‑grained access rights, integration with KMS (Key Management Systems), consolidated reports and statistics on backup and restore operations, compliance tools, and collaborative backup management for multiple users within the same entity.

Core components such as Ptar, Plakar agent, and Kloset are and will remain licensed under the ISC license. The Plakar UI, while part of the Enterprise Edition, will continue to be distributed with the agent and will remain free to use for managing a single Kloset.

Our commitment is to maintain accessibility for small organizations, while ensuring a sustainable business model that balances Plakar Korp’s need for profitability with our mission to keep the solution open and widely available.
